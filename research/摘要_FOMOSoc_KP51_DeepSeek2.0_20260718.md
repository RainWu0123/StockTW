## 核心主張

**Kimi K3 出來不會讓 AI 支出縮水，反而會擴大。最終受益的是雲端巨頭，不是模型公司。**

主線拆成三層：
1. 模型層壓力：開源打閉源的定價權
2. 算力層擴張：領跑者壓力 + 傑文斯悖論
3. 系統層控制權：雲端平台與裝置入口才是真護城河

## 主題一：Kimi K3 是 DeepSeek 2.0 嗎

### 事實層

- 月之暗面（Moonshot AI）旗艦 Kimi K3，2.8 兆參數
- MoE 架構：896 個專家，每次激活 16 個
- 100 萬 token 上下文，主打 Agent 能力
- 網頁前端代碼實測打贏 Claude Fable 5 和 GPT-5.6 Sol
- Bloomberg 下標「新的 DeepSeek 時刻」

### 帳面便宜實際不便宜的貓膩

- API 定價：每百萬 token 輸入 3 美元、輸出 15 美元
- 帳面跟 GPT-5.6 Terra 差不多，比 Claude Fable 5 便宜一大截
- Artificial Analysis 實測：預設開最大推理模式，output token 是其他模型平均 2 倍
- 折算實際任務成本跟 GPT-5.6 Sol 差不多
- 月之暗面說未來會開放讓用戶調推理深度，性價比才會真正釋放
- **真實指標：每美元能買多少有效智慧**

### 跟 DeepSeek 的關鍵差異

- DeepSeek 走極致價格破壞
- Kimi K3 定價顯著高於國內同行，硬碰硬做大體量旗艦
- 中國模型第一次在特定實戰場景展現一線競爭力，不是只當便宜替代品
- **七月底完全開源權重**，2.8 兆參數頂尖模型變成免費資源，倒逼閉源巨頭

### Dylan Oh 補充（留言區 KP 認可）

- MoE 很多人都用，Moonshot 優化到極致
- quantile balancing 增加專家調度穩定性
- 開發自己的 Kimi Delta Attention，混合式 attention 設計，在準確度與計算效率間取平衡
- 晶片出口限制反而催生更強技術，某種程度因禍得福

## 主題二：開源對 OpenAI/Anthropic 的殺傷

### KP 的邏輯鏈

1. 過去開源模型跟頂尖閉源差半代到一代
2. Kimi K3 歷史上第一次接近頂尖水準 + 百萬上下文 + 原生視覺的巨無霸送進開源
3. 需要機密數據的政府和金融機構，突然有不用依賴美國雲端 API 的頂級選項
4. 閉源護城河「最聰明所以高價」被動搖

特別是 Anthropic，以「寫程式最強」當賣點，壓力最大。

### 雲端巨頭樂見模型碎片化（KP 最有意思論點）

微軟、亞馬遜、Google 表面投資 OpenAI 和 Anthropic，心底不希望閉源獨大。若只剩一兩家閉源壟斷，雲端巨頭被邊緣化成水電工，最肥的利潤和客戶關係被模型公司拿走。

他們要的劇本：**天下大亂、群雄割據**。5-10 個實力相當模型混戰，企業客戶選擇困難，雲端平台變成「模型調度與管理」關鍵節點。亞馬遜 AWS 對開源模型支援最積極，就是這個邏輯。

## 主題三：AI 支出會不會減少（KP 核心主張）

華爾街恐慌論述：Kimi 追上 GPT/Claude → AI 泡沫要破、算力需求到頂、OpenAI/Anthropic 被邊際利潤壓垮。

KP 打臉：**模型層利潤縮水，往往是基礎設施與上游應用迎來更健康爆發的前兆。**

### 三層邏輯

**第一層：領跑者壓力反加速燒錢**
- 領先者被追上，理性選擇不是放慢，是打興奮劑加速
- 為重新拉開差距，OpenAI/Anthropic 只能加速訓練更大更強下一代
- 軍備競賽停不下來，只要對手逼近，研發訓練算力只會增加不會減

**第二層：有能力燒（雲端 credits 閉環）**
- 微軟、亞馬遜、Google 投資很大一部分是雲端算力額度（Cloud Credits），不是現金
- 實驗室用掉額度 → 變成 Azure/AWS 雲端營收
- 閉環降低巨頭實際現金壓力，還撐大雲端業務版圖
- 對巨頭是黃金平衡點：前沿實驗室有錢推進 + 純模型層面臨競爭壓力 + 邊際利潤被壓縮 + 雲端吞下產業鏈最大最穩定的蛋糕

**第三層：傑文斯悖論**
- token 價格暴跌 50%，企業反應不是「省一半預算」，是「用同樣預算塞進以前不敢試的業務場景」
- 多模型協作、長思考鏈、AI 代理反覆對話自我糾錯，token 消耗成百上千倍
- 單個 token 利潤變薄，整體推理算力需求迎來更巨大爆發

### 結論

- 產業步入成熟
- 價值向上下游轉移：算力基礎設施 + 編排層工具鏈吃下更多經濟剩餘
- 重心從「哪家模型最強」轉向「如何靈活組合不同模型解決真實問題」
- 類比網路早期：基礎設施夠便宜後，瀏覽器不再是唯一獲利來源，催生搜尋引擎和電商

## 主題四：微軟的逆向資訊悖論

### 逆向資訊悖論

- Arrow 原本資訊悖論：承擔風險的是賣方
- Nadella 的逆向資訊悖論：AI 時代承擔風險變成買方
- 企業員工用 AI 修正錯誤答案：第一次付 token 費用，第二次把內部運作經驗免費送給 AI
- 長期下來模型廠商越來越懂你的行業，你對黑盒子一無所知，差距越拉越大

### Frontier Tuning

- 微軟在 Build 大會推出的企業客製系統
- 企業專屬 Azure 雲端邊界內建私有強化學習環境（RLE）
- 導入企業專屬數據（SharePoint、Microsoft 365）
- 記錄員工真實決策與糾錯軌跡
- 優化後模型參數完全保留在企業 Azure 環境
- 早期客戶 Land O'Lakes：複雜報告準確度提升至九成以上，token 消耗效率提高近十倍

### KP 的戰略解讀

- 微軟把商業模式從低毛利 token 銷售升級成系統平台
- 企業一旦把核心學習閉環架在 Azure 上，未來極難轉移
- 降低對單一模型依賴，「底層模型是誰不重要，在 Azure 運行的私有閉環才關鍵」
- 削弱其他模型廠商談判籌碼

**KP 金句**：模型是流水的兵，Azure 才是微軟鐵打的營盤。

與 Palantir Alex Karp 之前的「企業正把核心流程和專利數據免費送給前沿模型實驗室」遙相呼應，微軟和 Palantir 在聯手建立「幫企業奪回 AI 控制權」新共識。

## 主題五：蘋果告 OpenAI 的入口之爭

### 事實

- OpenAI 2025 年用約 65 億美元收購 io Products（Jony Ive 創立的硬體新創）
- 挖角超過 400 名蘋果硬體團隊成員
- 蘋果訴狀細節包括面試中被要求展示未公開原型零件

### KP 的判斷

- OpenAI 從「軟體互補品」變成「硬體替代品」
- OpenAI 能做出 AI 原生硬體載體（智慧眼鏡或新型隨身終端），用戶對 iPhone 作為 AI 入口的依賴會被大幅削弱
- 蘋果視硬體為生命線，這條紅線絕不容許踩

### 老巨頭的地盤默契 vs 新貴的版圖重塑

- 蘋果和 Google 是老巨頭，各有不可動搖地盤
- Google 每年付蘋果數百億美元維持 iPhone 預設搜尋引擎，相互保證毀滅成本高，傾向合作共生
- OpenAI 是沒有歷史包袱的新貴，想重塑版圖
- OpenAI 意識到：單純擁有模型層不夠，沒有自己的分發通路，大模型隨時可被替代，淪為巨頭生態打工仔

### 系統層控制權 > 模型層

- 雲端模型再聰明，必須透過物理載體跟用戶互動
- 控制裝置 → 控制注意力、上下文數據、流量分發權
- 時間賽跑：OpenAI 必須在模型溢價消失前做出硬體，蘋果用法律訴訟極大化開發成本拖慢它
- 全棧巨頭終極壁壘：蘋果和 Google 從晶片設計、硬體製造、OS、分發通路到用戶基數每層築壁壘
- 演算法可以速成，數十年累積的硬體與生態帝國不容許任何人輕易染指

## 主題六：CoreWeave 做空美光

### 事實

- Reuters 2026-07-14 報導
- CoreWeave 正在與投銀研議，考慮買進美光和 SK 海力士的 put option
- AI 雲端服務商反手做空自己核心供應商

### 邏輯

- 美光近期跟客戶簽三到五年 LTA（長期供應合約）
- LTA 有「價格下限」條款：即使未來現貨價格回檔，買方仍必須依約按最低保證價採購
- CoreWeave 擔心記憶體暴跌時被長約綁死，必須繼續高價採購 → 毛利率受損 + 庫存跌價損失

### 為什麼是 CoreWeave 做這事

- 創辦人 Michael Intrator、Brian Venturo、Brannin McBee 有大宗商品和能源交易背景
- 曾管理天然氣對沖基金
- 對他們來說 GPU 和記憶體晶片本質上是大宗商品，算力中心是加工廠
- 用衍生性商品對沖晶片成本，完全符合人設

### 類比的局限性

- 跟航空業用原油期貨鎖燃油成本類似
- 但航司用直接對應的原油期貨，CoreWeave 用個股 put，股票價格跟實際晶片價格有落差，避險不精準
- 原油市場規模龐大，個股衍生性商品流動性有限，大額避險成本極高
- **暴露半導體產業鏈長期空白：晶片有大宗商品波動特性，卻沒有像原油黃金那樣高流動性、能直接對沖實物價格的標準化衍生品**

### 對供應商的影響（KP 微妙判斷）

- 正面：買方認真對待長期合約，不是等價格暴跌才賴帳毀約
- 如果 CoreWeave 在金融市場賺到錢，能心平氣和繼續履行高於市價採購合約，增加記憶體大廠營收確定性
- 負面：市場傳出針對美光的大額 put 買單，等於宣告「大客戶預期晶片將大跌」，景氣循環末端壓力加重

### KP 收斂

- 半導體景氣繁榮頂峰往往伴隨繁華落盡後買方大規模賴帳毀約砍單
- CoreWeave 主動用金融避險，反而展現成熟的商業契約精神
- 如果試驗成功，會為半導體供應鏈提供新範本：用衍生性金融商品當緩衝墊，讓買賣雙方面對週期時都能體面活下來

## 對近期研究的含義

1. **CoreWeave 做空美光 = MS TMT 動能洗盤的另一面**：大客戶用衍生品對沖記憶體崩跌風險，是籌碼面訊號。短線賣壓在定價未來下跌，但長約綁住供給確定性。long basket 基本面仍強，對照 CoreWeave LTA 還在簽，產業層健康，槓桿層在去化。

2. **KP 對「AI 支出不會減」的論證直接支持台積 7/16 法說的 Q2 營收破表**：領跑者加速燒 + 傑文斯悖論 + 雲端 credits 閉環 → 算力需求中期沒問題。台積 2026 capex 調高至 640 億美元是同一條線。

3. **開源打閉源對台股影響有限**：KP 講的是模型層和雲端層價值分配，對台積、HBM、先進封裝這些算力基礎設施供應鏈，反而是需求擴大的受益者。不管 Kimi 還是 GPT 贏，算力都要燒。

4. **蘋果告 OpenAI 對 Apple 供應鏈有關**：OpenAI 做硬體的難度被蘋果用法律和生態壓制，裝置層控制權壁壘比模型層高。台積做 Apple 晶片的訂單確定性，跟系統層控制權的持久性綁在一起。

## KP 的開場心法（值得抄）

- 預測高低點並非在市場賺錢的依靠，價值在市場劇烈波動大家恐慌時能清清楚楚看懂現在的真實狀況
- 電子報點閱率下滑是反向指標：市場暴跌時人腦防禦機制逃避痛苦，關掉看盤軟體，停止閱讀深度分析
- 不學習的代價是把決策權交給市場上最恐慌的情緒
- 市場大跌時才是研究 ROI 指數級上升的時刻
- 砍掉（基本面確實轉壞）/ 保留（核心競爭力無可撼動只是被情緒錯殺）/ 加碼（長期趨勢明確市場送禮）
