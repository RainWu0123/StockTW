# Round 3 — 風控專員修正式立場回應

**Persona：** 風控專員  
**Round 1 主張：** 1) 台股特有風險特徵 2) max drawdown 損失函數 3) 台股制度 RL 模擬環境  
**Round 2 被質疑：** (a) 技術分析師指出 max drawdown 在漲跌停下失準、RL state space 不足 (b) 籌碼分析師指出 RL 與風控存在天然衝突

---







## 一、堅持不變

1. **Max Drawdown 仍是台股風控的核心損失函數**  
   VaR 在台股肥尾市場（處置股、地緣政治、半導體制裁）嚴重低估極端虧損；Sharpe Ratio 在泡沫與恐慌期會因波動放大而失真。Max DD 是唯一能同時捕捉「連續下跌」「跳空斷板」「強制平倉連鎖」的單一指標，且與投資人實際體驗的虧損感受一致。

2. **RL 模擬環境是處理「狀態相依風險」的唯一可行路徑**  
   台股的融資維持率、處置股票、散戶追漲殺跌，都會形成「個股下跌 → 融資斷層 → 進一步下跌」的反饋循環。傳統歷史回測是靜態的，無法建模這種動態交互；RL 的馬可夫決策框架（state → action → reward → next state）是唯一能模擬這種交互的架構。

3. **台股四大制度特徵必須內建於任何量化模型中**  
   漲跌停（10%/15%）、融資融券（維持率、強制平倉）、處置股票（成交量限額、波動限制）、除權息（價格跳空＋融資擔保品減值），這是台股與美股的結構性差異。忽略這些特徵的模型，本質上是在「美股環境」訓練、在「台股環境」部署，必然會災難性失效。

---







## 二、根據反駁調整的

### 1. 針對「max drawdown 在漲跌停下失準」的修正

**技術分析師的核心質疑：** 台股漲跌停制度會截斷連續價格路徑，使得 drawdown 分布呈現「跳躍式」而非「連續滑點」，傳統 Max DD 計算會低估實質虧損。

**我的調整：**
- **雙軌 Max DD 計算框架**：
  - **名目價格 DD（Nominal Price DD）**：維持傳統計算方式（基于日低價），用於監控投資組合的名目水位。
  - **可實現損失 DD（Realizable Loss DD）**：考慮漲跌停限制與流動性。若連續停板，以「前一個可交易日均價」作為估算賣出價；若為小盤流動性枯竭，以「前 five 日均量 × 10%」作為最大可實現賣出比例估算。
  - **強制分離「市場風險 DD」與「流動性/制度 DD」**：當個股被列為處置股票、或融資維持率低於 130% 時，自動切換到可實現損失 DD 計算，不再用連續價格推估。

- **動態壓力上限（Dynamic DD Ceiling）**：  
  Max DD 不再是一個靜態常數（如 20%）。當先行指標觸發時，自動下調：
  - 融資大戶餘額周轉率 > 3σ → DD 上限 -5%
  - 外資連續 5 日賣超 + 散戶融資增 → DD 上限 -10%
  - 個股被列處置 + 三大法人群體反向 → DD 上限 -15% 或直接暫停交易

### 2. 針對「RL state space 不足」的擴充

**技術分析師指控：** 只考慮 OHLCV + 技術指標，忽略了融資維持率、處置股、外資動態、匯率。

**我的調整 — 5 層 State Space 設計：**
| 層級 | 內容 | 目的 |
|------|------|------|
| L1 價格層 | OHLCV + 20/60 日均線斜率 + 動量指標 | 捕捉價量結構 |
| L2 籌碼層 | 三大法人買賣超、融資融券餘額、借券餘額、融資維持率分布 | 捕捉法人動能與散戶槓桿 |
| L3 制度層 | 處置股票標記、漲跌停狀態、除權息日、可交易額度限制 | 內建台股交易規則 |
| L4 宏觀層 | 台幣匯率、美國 10 年期殖利率、VIX、DRAM 現貨價 | 捕捉外部風險因子 |
| L5 微結構層 | 散戶/法人成交比、融資大戶集中度、當沖成交比 | 捕捉市場微結構與散戶行為 |

**時序處理要點：**
- 法人數據採「動能加速度」（今日買超 - 昨日買超）而非絕對值，回應籌碼分析師指出法人領先期只有 1-3 日的问题。
- 所有法人/融資數據強制 T+1 標註，杜絕 lookahead bias。

### 3. 針對「RL 與風控天然衝突」的架構修正

**籌碼分析師的警告：** Event-Driven RL 傾向最大化 reward，在法人買超泡沫中會不斷加碼，直到融資余額天花板 + 法人轉賣的 crash 來臨。

**我的調整 — 雙 Agent 協作框架：**
- **交易 Agent（Trader）**：負責選擇個股、部位大小、進出場時機，目標是最大化經風險調整後的收益（CVaR-constrained Sharpe）。
- **風控 Agent（Risk Guardian）**：擁有「熔斷權」：
  - 監控交易 Agent 的即時部位集中度、名目 DD、可實現 DD。
  - 當觸發動態 DD 上限時，可直接要求減碼或清倉。
  - 當偵測到「法人同步轉賣 + 融資斷層 + 處置股」三重訊號時，直接中斷交易 Agent 的行為。
- **Reward 設計**：
  - 主 Reward：`CVaR(95%) - λ × Max DD`
  - 次 Reward：風控合規度獎勵（未觸發熔斷且部位可控）
  - 懲罰項：動態 DD 超限、融資過度集中、處置股持有









## 三、整合其他兩位優點後的強化版方案：「三層防線 + 雙軌 RL」

### 第一層：資料品質與籌碼時序標註（整合籌碼分析師）

| 要素 | 做法 | 解決的問題 |
|------|------|------------|
| 法人數據時序標註 | 計算連續買賣超天數、動能加速度、板塊同步係數 | 法人領先期短、相位問題 |
| 上市/上櫃/興櫃分群 | 三市場分別建模，拒絕混幣訓練 | 披露規則不同、散戶比例差異 |
| T+1 延遲標註 | 所有法人特徵延遲一個交易日納入 | 杜絕 lookahead bias |
| 借券/重複申報去重 | 交叉比對外資買超與借券賣出，偵測假訊號 | 資料品質致命傷 |

### 第二層：價量結構先行指標（整合技術分析師）

| 要素 | 做法 | 與風控的銜接 |
|------|------|-------------|
| 月線/季線斜率 | 跌破月線時，動態下調 DD 上限 5-10% | 技術轉折 = 風控警戒 |
| 量價背離 + 法人加速度 | 法人買超但量價背離 → 預警散戶出貨 | 籌碼反轉先行指標 |
| 均線系統作為動態 DD 參考 | 股價與月線乖離率 > 20% 時，自動要求部位減碼 | 價位偏離 = 風險累積 |

### 第三層：雙軌 RL 風控執行層（風控專員核心）

**環境設計：**
- 5 層 State Space（價格 / 籌碼 / 制度 / 宏觀 / 微結構）
- **制度規則強制執行**：漲跌停價格限制、處置股成交量限制、融資上限動態計算。

**損失/獎勵架構：**
- 日頻損失：`L = - Sharpe + α × Max DD + β × 管理成本`
- 風險約束：`CVaR(95%) ≤ 2%`，`名目 DD ≤ 動態上限`，`可實現 DD ≤ 動態上限 × 1.2`
- 熔斷機制：風控 Agent 在觸發條件下有強制清倉權。

**可執行驗證方案（回應質疑匯總清單）：**

| 質疑點 | 驗證方法 | 預期結果 |
|--------|----------|----------|
| 漲跌停截斷 DD | 以 2019-2024 年處置股/漲跌停日重構價格序列，計算「連續 DD」vs「跳躍 DD」分布 | 可實現 DD 應顯著高於連續 DD，验证截斷效應 |
| RL state space 不足 | ablation study：僅用 OHLCV vs 加入 5 層特徵，比较 out-of-sample Sharpe | 5 層版本應顯著優於僅價格版本 |
| RL 與風控衝突 | 單一 Agent（max return）vs 雙 Agent（交易+風控）對比 | 雙 Agent 的 Max DD 應顯著降低，且 Sharpe 不會大幅衰退 |
| 法人時序 | 計算各法人/大盤的 cross-correlation，觀察滯後係數隨市場狀態切換 | 外資領先期約 1-3 日，自營商/投信滯後可達 5-7 日 |
| 散戶/法人對立 | 計算「法人淨買超」與「散戶融資增減」的相關係數，再與極端回撤事件做對應 | 散戶融資大增 + 法人賣超組合，應預signal 未來 5-10 日的極端 DD |







## 四、結論：我的修正式立足點

- **對技術分析師的共識**：價量結構（月線季線、量價背離）確實是法人籌碼的「價格先行指標」，且不受 T+1 揭露延遲。我接受將月線斜率、量價結構作為「動態 DD 上限」的觸發條件，這能讓風控更即時。
- **對籌碼分析師的共識**：法人數據的時序問題、資料品質問題、上市櫃分群，確實是 RL 成敗的基礎。我接受 T+1 延遲標註、法人動能加速度、借券去重，作為 RL state space 的強制輸入。
- **我的核心堅持**：風控不能只靠「事後 Max DD」；必須透過「動態壓力上限 + 雙軌 RL + 風控 Agent 熔斷權」三合一架構，實現「事前預防、事中控制、事後修正」的完整閉環。

**沒有人可以單獨打赢這場仗。**  
- 技術分析師提供價量結構的先行預警。  
- 籌碼分析師提供法人籌碼的時序標註與資料品質把關。  
- 風控專員提供 Max DD 損失函數與 RL 模擬環境的執行框架。  

三層防線的整合方案，才是台股 AI 交易的可行路徑。
