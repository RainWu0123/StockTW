# Round 2 — 籌碼分析師反駁技術分析師的事件驅動 RL 方案

**Persona：** 籌碼分析師  
**立場：** 法人連續訊號時序標註訓練集、族群輪動+籌碼共振、資料品質優先

---

## 技術分析師主張摘要
技術分析師主張以「事件驅動強化學習（Event-Driven RL）」搭配「法人題材」作為台股分析AI的核心架構，意圖讓agent在事件觸發後動態調整部位。

---

## 具體反駁

### 1. 法人題材在台股本質上是「落後的」，RL 拿落後特徵做即時決策只會放大追高殺低
- 三大法人的日買賣超資料是「收盤後揭露、T+1才生效」。技術分析師的 RL 如果以「法人買超題材」作為 state feature，agent 在事件當日（收盤前）根本拿不到這個特徵；若用預估值或盤中估算，只會引入巨量噪聲。
- 即使拿到「隔日生效」的法人數據，法人籌碼在台股的有效領先期通常只有 **1-3 個交易日**，超過這個窗口後，法人買超往往是「拉高給散戶看」的誘多訊號。
- Event-Driven RL 的 reward 設計若以「事件發生後N日報酬」計算，會把法人題材的**反向欺騙**誤判為 valid signal，導致 agent 習得「看到法人買超就做多」的錯誤策略，在實盤上會持續撞牆。

### 2. 事件驅動 RL 的 episode 邊界設計，在台股根本無法乾淨定義
- Event-Driven RL 依賴「事件觸發 → 持有 → 事件結束 → 平倉」的 episode 結構。但台股的事件是**重疊且連續**的：
  - 一個月內可能同時存在「法人大買超 + 題材熱點 + 融資余額創高 + 董監事質押警示」。
  - 這些事件的時間尺度不同：法人連續買超可能持續 20 天，而融資 ceiling 警戒可能只給 3 天。
- 技術分析師若只抓單一事件（例如「突破 + 法人買超題材」）當 episode boundary，會遺漏同時發生的反向訊號，導致環境的 reward 嚴重偏誤。

### 3. 法人題材 + RL 會遇到「稀疏 reward + 過擬合歷史題材」的雙重陷阱
- 台股熱點族群輪動極快（半導體 → AI → 重電 → 內需 → 生技），技術分析師若把某一年度的法人題材 event 直接餵進 RL，模型只會背下那一年的族群節奏，遇到隔年的題材切換時策略直接失效。
- Event-Driven RL 在 sparse reward（很多事件後沒有顯著報酬）的情況下，需要極大的樣本數才能收斂。但台股的法人題材歷史樣本極少（一年下來「連續法人買超 + 股價突破」的乾淨 episode 可能不到 100 筆），根本不足以支撐 RL 訓練。

### 4. 技術分析師忽略了「資料品質」是法人題材的致命傷，而非架構问题
- 我的立場明確要求「實收法人數據、避免 lookahead、區分上市櫃」。技術分析師的 Event-Driven RL 若直接用券商點速或盤中估算法人數據，會把錯誤的法人方向餵進 RL 的 state space，agent 學的是**預估錯誤的法人人數**，而非真實籌碼。
- 上櫃/興櫃與上市的法人披露規則、股權分散度、散戶比例完全不同，混在一起訓練只會產出一個在所有市場都適用的無用策略。

### 5. 族群共振的「同步崩盤」訊號，RL 獎勵函數極難捕捉
- 籌碼分析的核心是「共振」：同一板塊多檔個股同時出現融資余額暴增 + 法人同買，這往往是族群主升段的上半場；但當「同步賣超 + 融資斷層」同時出現時，是多空反轉的關鍵。
- Event-Driven RL 以單一個股 episode 為單位，根本看不到板塊層級的共振結構。技術分析師若要 RL 自己「發現」板塊共振，需要 multi-agent 或 graph-based RL，這已遠超他簡單陳述的範圍，且 training complexity 是指數級上升。

### 6. 風控與 RL 的天然衝突：Event-Driven RL 傾向最大化 reward，容易在法人題材泡沫中爆倉
- 在法人連買、散戶瘋狂進場的行情末端，Event-Driven RL 若只看股價突破 + 法人題材的正 reward，會不斷加碼，直到「融資余額天花板 + 法人轉賣」的 crash 來臨。
- 籌碼分析師認為，這種風險應該在**資料集設計（時序標註）與損失函數（加重反轉情境權重）** 時就預先遏止，而不是寄望 RL 自己學會保命。

---

## 結論：技術分析師方案的致命錯誤
| 問題 | 台股實務結果 |
|------|--------------|
| 法人數據落後一期 | RL state 含未來資訊或預估值 noise |
| 事件 episode 邊界不清 | reward 嚴重偏誤 |
| 歷史題材稀疏 | RL 過擬合過去族群，失效快 |
| 忽略上市櫃差異 | 策略泛化能力差 |
| 無板塊共振机制 | 忽略族群輪動關鍵風控點 |
| RL 與風控衝突 | 泡泡行情中歸零風險高 |

**我的堅持不變：**
1. **先建法人籌碼時序標註資料集**（連續天數、動能加速度、板塊同步係數），這是所有模型的基礎。
2. **資料品質優先**：實收數據、T+1 標註、上市櫃分群，否則任何 RL 都是垃圾進垃圾出。
3. 就算最終要上 RL，也該先在**乾淨的籌碼特徵空間**上做，而不是拿落後的事件驅動 signal 直接訓練。
